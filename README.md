# Telecom Channel Saturation Forecasting

This repository combines an Airflow-based data pipeline with a machine learning workflow to detect and predict telecom channel saturation risk. The project is designed for operational network use cases where early detection of high utilization can help prevent performance degradation, outages, and capacity planning issues.

## Overview

The solution is organized in two main components:

- Airflow pipeline: ingests traffic and monthly master data, builds the unified master dataset, and produces processed parquet outputs for downstream analytics.
- Machine Learning pipeline: creates the target label for future saturation events, trains and evaluates models, and generates metrics, figures, and predictions.

## Key capabilities

- Orchestrates monthly and incremental data processing with Airflow.
- Builds and enriches the master reference dataset for telecom channels.
- Produces structured parquet outputs suitable for ML consumption.
- Trains and compares classification models for saturation risk prediction.
- Generates business-oriented artifacts such as metrics, confusion matrices, calibration reports, and predictions.

## Prerequisites

Before running the project, make sure you have:

- Python 3.10+ (recommended)
- Docker Desktop with Docker Compose
- Access to the input files expected by the pipelines

## Quick start

### 1) Run the Airflow workflow

From the Airflow folder:

```bash
docker compose up -d
```

Then open the Airflow UI in your browser:

- http://localhost:8080

Default login credentials used in the compose configuration:

- Username: admin
- Password: admin

### 2) Run the machine learning pipeline

From the Machine_Learning folder:

```bash
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# or .\.venv\Scripts\Activate.ps1  # Windows PowerShell
pip install -r requirements.txt
python run_pipeline.py --mode ml
```

Other useful commands:

```bash
python run_pipeline.py --mode eda
python run_pipeline.py --mode qa --qa-mode full --strict-qa
python run_pipeline.py --mode full
```

## Data expectations

The Airflow workflow expects input data in the following places:

- Airflow/data/raw/traffic/: traffic files
- Airflow/data/raw/mensual/: monthly master files
- Airflow/data/mapping/: reference and mapping files

The machine learning pipeline uses the processed outputs generated by the data workflow and writes artifacts under:

- Machine_Learning/outputs/eda/
- Machine_Learning/outputs/ml/

## Configuration

- Airflow configuration lives in Airflow/pipeline/config.py.
- Machine learning settings are defined in Machine_Learning/config/config.yaml.

Key ML settings include:

- risk threshold for defining saturation events
- prediction horizon in days
- classification threshold for final decisions
- model tuning parameters and cost assumptions

## Outputs

### Airflow outputs

- Parquet files for enriched traffic data
- Unified master datasets for downstream analytics
- Intermediate state and processed data artifacts

### Machine Learning outputs

- Trained models and preprocessing artifacts
- Prediction files and evaluation metrics
- Plots and comparison reports for model analysis
